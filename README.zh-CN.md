# AndroidClaw

[English README](README.md)

> 当前仓库采用了 **完整的 vibe coding 工作流**：从需求整理、架构决策、代码实现、调试到文档沉淀，都通过 AI 协同编码在手机端场景中快速迭代完成。

AndroidClaw 是一个以 **LLM 驱动手机操作** 为核心目标的 Android 原生智能体项目。它把大模型的任务规划能力与 Android 无障碍能力结合，让模型能够理解用户意图并执行真实手机操作。

## 项目灵感来源

本项目在设计思路上参考了多个优秀项目和产品，包括：

- OpenClaw
- NanoBot
- 豆包手机助手
- AutoGLM

项目目标不是复刻某一个实现，而是将其中可落地的思路沉淀成一个更适合 Android 端运行、扩展与部署的工程。

## 核心定位

让 **LLM 成为手机操作员**：

- 用自然语言理解用户需求
- 自动规划执行步骤并调用工具
- 读取当前界面信息并执行手势操作
- 以常驻守护进程方式保持可用性和稳定性

## 技术架构

AndroidClaw 当前主要分为以下层次：

- **Agent Core 层**：多轮规划执行、记忆压缩、工具调用、子代理与技能机制
- **Tool 层**：手机/文件/命令/网络工具，MCP 外部工具桥接，技能工具
- **UI 理解层**：优先无障碍结构化快照（`phone_get_ui_snapshot_compact`），截图/文本兜底
- **Channel 层**：统一消息通道抽象（本地对话通道 + 飞书通道）
- **存储与稳定性层**：Room 持久化、守护服务、健康检查、运行日志

## 当前已支持功能

### LLM 与 Agent 能力

- 可配置模型提供方与模型参数
- Agent 会话管理与记忆
- 工具注册、路由与执行框架
- 执行耗时、迭代次数、Token 消耗等运行元信息记录

### 手机操作能力（重点）

- 基于无障碍服务的 UI 交互
- 按文本点击 / 长按
- 按坐标点击 / 长按
- 基于结构化节点的点击与输入：`phone_click_node` / `phone_input_node`
- 基于截图坐标映射点击：`phone_click_coordinates_from_screenshot`
- 滑动与滚动操作
- 全局按键：返回 / Home / 最近任务
- 按包名打开应用
- 读取当前屏幕文本与窗口标题
- 截图能力（data URL + 文件落盘调试）

### 文件与命令工具

- 文件读写、列举、删除、建目录（应用范围）
- 通过工具层执行命令

### 守护与稳定性

- 前台守护服务
- 开机自启动
- WorkManager 健康检查与自愈
- 网络变化后的自动恢复流程

### 远程交互集成

- 飞书机器人接入
- 基于 WebSocket 的消息处理
- 通道抽象层复用消息收发链路

### 应用侧体验

- 首次启动权限引导页
- 必需权限未满足时禁止进入主页
- 统一设置页（三段式）：模型设置 / 飞书设置 / Agent 设置
- 执行悬浮窗（思考/工具状态）、暂停恢复控制、执行过程消息
- 图片本地持久化与可缩放预览
- 运行日志查看与长按删除

## 技术栈

- Kotlin + Android SDK（minSdk 24, targetSdk 34）
- Room、WorkManager、Coroutines
- LangChain4j（模型与工具集成）
- OkHttp、Gson

## 构建方式

./gradlew assembleDebug
